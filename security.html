Current security limitations — read before relying on Vestora

Early access means honesty about what is not finished yet. The full detail lives in the security architecture.

  • !The main database is not yet encrypted at rest by Vestora. It stays on your device; enable BitLocker / Windows Device Encryption. App-level encryption is scheduled on the roadmap (Q3 2026); the mobile apps now in internal testing already encrypt their database at rest.
  • !The early-access installer is not yet code-signed. Only run a build received directly from us; we confirm SHA-256 checksums by email.
  • !No independent security audit has been completed yet. One is planned before wide release; results will be published.
  • !Backups are your responsibility. Vestora provides encrypted backup tools — please use them, and protect your device like the vault it is.
